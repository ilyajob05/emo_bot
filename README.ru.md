# Emotional De-escalation MCP Server v2

MCP-сервер для **стратегического управления диалогом** и эмоциональной деэскалации в ботах поддержки клиентов.

Основная проблема: боты поддержки застревают в циклах — спрашивают номер заказа 5 раз, повторяют одно и то же извинение, игнорируют сигналы эскалации. Этот сервер обнаруживает такие паттерны и подсказывает боту **что делать дальше** — сменить стратегию, перевести на живого оператора или перестать повторяться.

**Ключевая возможность: детерминированное обнаружение эскалации.** Инструмент `strategy_suggest` распознаёт 7 проблемных паттернов (повторяющиеся вопросы, угрозы судом, сигналы оттока, запрос оператора, эмоциональная эскалация, отсутствие прогресса, повторные обращения) и рекомендует конкретные действия — включая момент, когда нужно перевести на живого оператора. Работает без вызовов LLM, офлайн, мгновенно.

Основная публикация: [Building an Emotional Guardian for AI Chatbots](https://medium.com/@ilyajob05/building-an-emotional-guardian-for-ai-chatbots-7e7bd0340e96)

Основано на: [Ошибки, которых следует избегать при разработке чат-ботов для поддержки пользователей](https://medium.com/@ilyajob05/mistakes-to-avoid-when-developing-chatbots-for-user-support-5eefa21256ab)

## Стратегическое управление диалогом

MVP покрывает 80% проблемных сценариев поддержки **без каких-либо вызовов LLM**. Детерминированный движок использует лемматизацию spaCy и сопоставление паттернов для обнаружения проблем и рекомендации стратегий:

### Обнаруживаемые паттерны

| Паттерн | Что обнаруживает | Эскалация |
|---------|-----------------|-----------|
| **repeated_question** | Бот задал один и тот же вопрос 2+ раза | После 4 повторений |
| **legal_threat** | Пользователь упоминает суды, регуляторов, адвокатов | Немедленно |
| **human_request** | Пользователь явно просит живого оператора | Немедленно |
| **churn_signal** | Пользователь угрожает отменить, уйти или требует возврат | Через 1 ход |
| **emotion_escalation** | Эмоциональная интенсивность пользователя растёт | Через 2 хода |
| **no_progress** | Обе стороны повторяются, диалог застрял | После 4 застойных ходов |
| **repeated_contact** | N-е обращение пользователя за сегодня (через метаданные) | После 3-го обращения |

### Эскалация на живого оператора

Сервер предоставляет чёткие, действенные сигналы эскалации:

- **`should_escalate_now: true`** — перевести немедленно (угрозы судом, явный запрос оператора, 4+ повторных вопроса, 3+ обращений за день)
- **`escalate_after_n_more_turns: N`** — эскалировать, если не решено в течение N ходов
- **Система приоритетов** гарантирует, что самый критичный паттерн определяет стратегию: legal_threat > human_request > repeated_contact > repeated_question > no_progress > churn_signal > emotion_escalation

### Антипаттерны

Движок отслеживает, что бот уже говорил, и генерирует правила «НЕ делай»:
- «НЕ спрашивать номер заказа снова» (если спрашивал 2+ раз)
- «НЕ использовать "Я понимаю ваше разочарование" — уже сказано 2 раза»
- «НЕ спорить с клиентом о его правах» (при угрозах судом)
- «НЕ говорить "успокойтесь"» (при эмоциональной эскалации)

## 5-осевая модель стиля коммуникации

Каждое сообщение характеризуется вектором стиля — 5 независимых осей на дискретной шкале от **-2** до **+2**:

| Ось | -2 | -1 | 0 | +1 | +2 |
|---|---|---|---|---|---|
| **W** Теплота | холодный, отстранённый | прохладный | нейтральный | дружелюбный | тёплый, эмпатичный |
| **F** Формальность | сленг, грубость | неформальный | сбалансированный | профессиональный | формальный, официальный |
| **P** Игривость | мертвенно серьёзный | сухой | сбалансированный | лёгкий, остроумный | юмористичный, ироничный |
| **A** Напористость | неуверенный, робкий | осторожный | сбалансированный | уверенный | требовательный, напористый |
| **E** Экспрессивность | краткий, сдержанный | скупой | сбалансированный | живой | эмоциональный, интенсивный |

### Комбинации стилей

| Стиль | W | F | P | A | E |
|---|---|---|---|---|---|
| Сарказм | -2 | -1 | +2 | 0 | +1 |
| Дружелюбный юмор | +1 | -1 | +2 | -1 | 0 |
| Флирт | +2 | -2 | +1 | -1 | +1 |
| Деловой тон | 0 | +2 | -2 | 0 | -1 |
| Агрессия | -2 | -2 | -2 | +2 | +2 |
| Отчаяние | 0 | -1 | -2 | -1 | +2 |
| Пассивная агрессия | -1 | 0 | +1 | +1 | -1 |

## Стратегия деэскалации

Вместо одного коэффициента деэскалация работает **поосно**:

| Ось | Сдвиг | Обоснование |
|---|---|---|
| Теплота | +1 | Повысить эмпатию |
| Формальность | +1 | Чуть более профессионально |
| Игривость | → 0 | Снизить риск сарказма |
| Напористость | -1 | Снизить давление |
| Экспрессивность | -1 | Успокоить интенсивность |

Это разрывает петлю положительной обратной связи, обеспечивая постоянный сдвиг вектора ответа бота в более конструктивную зону.

## Режимы движка

Сервер поддерживает два режима выполнения, управляемых через переменную окружения `EMOTION_MCP_MODE` или параметр `mode` при каждом вызове:

| Режим | Как работает | Нужен API-ключ | Стоимость |
|-------|-------------|----------------|-----------|
| **host** (по умолчанию) | Инструмент возвращает структурированный промпт → хост-LLM (Claude Desktop/Code/LM Studio) выполняет его | Нет | Бесплатно |
| **api** | Инструмент вызывает Anthropic API напрямую, возвращает разобранные результаты | Да | Платно |

В **режиме host** каждый инструмент возвращает единый самодостаточный промпт с полной 5-осевой моделью, правилами анализа, контекстом сессии и инструкциями для конкретной задачи. Хост-LLM выполняет всё за один проход.

В **режиме api** сервер делает собственные вызовы LLM и возвращает структурированные результаты (JSON или Markdown).

Для переключения в режим API:
```bash
# Через переменную окружения
export EMOTION_MCP_MODE=api

# Или параметром при вызове
{"text": "...", "mode": "api"}
```

## Инструменты

### Стратегические инструменты

#### `strategy_suggest`

**Вызывайте ДО составления ответа.** Анализирует паттерны диалога и рекомендует, что делать дальше. Полностью детерминированный — без LLM, работает офлайн, мгновенно.

```json
{
  "dialogue_history": [
    {"role": "user", "text": "Где мой заказ?"},
    {"role": "bot", "text": "Уточните номер заказа."},
    {"role": "user", "text": "Я уже говорил! Уточните по телефону!"},
    {"role": "bot", "text": "Подскажите номер заказа, пожалуйста."},
    {"role": "user", "text": "Да что с вами не так?! Позовите оператора!"}
  ],
  "available_actions": ["lookup_by_phone", "escalate_to_human"],
  "user_metadata": {"total_contacts_today": 2},
  "language": "ru"
}
```

Пример ответа:
```json
{
  "recommended_strategy": "comply_with_human_request",
  "reasoning": "Пользователь явно запросил живого оператора. Попытки продолжить диалог ботом ухудшат ситуацию.",
  "action_sequence": [
    {"action": "escalate_to_human", "priority": "required", "note": "Пользователь явно попросил оператора — выполнить без промедления."}
  ],
  "anti_patterns": [
    "НЕ спрашивать номер заказа снова — бот уже спрашивал 2 раза",
    "НЕ говорить 'успокойтесь' или 'не нервничайте'"
  ],
  "escalation": {
    "should_escalate_now": true,
    "escalate_after_n_more_turns": null,
    "reason": "Пользователь явно запросил живого оператора"
  },
  "detected_patterns": ["human_request", "repeated_question", "emotion_escalation"]
}
```

**Стратегии по паттернам:**

| Паттерн | Стратегия | Ключевое действие |
|---------|----------|-------------------|
| repeated_question | alternative_identification | Попробовать поиск по телефону/email |
| legal_threat | immediate_supervisor_escalation | Перевести на руководителя сейчас |
| human_request | comply_with_human_request | Перевести на оператора сейчас |
| churn_signal | retention | Предложить компенсацию, подключить менеджера |
| emotion_escalation | de_escalation | Замедлиться, валидировать эмоции |
| no_progress | break_deadlock | Подвести итог и эскалировать |
| repeated_contact | priority_handling | Подтвердить, приоритизировать |

### Инструменты анализа

Все инструменты анализа принимают необязательные параметры:
- `session_id` — для отслеживания эмоционального состояния между ходами
- `mode` — `"host"` или `"api"` (переопределяет переменную окружения)

### `emotion_analyze`

Анализирует сообщение → эмоция + вектор стиля + метка стиля.

```json
{
  "text": "ну конечно, ваш замечательный бот мне так помог, спасибо огромное",
  "language_hint": "ru"
}
```

Возвращает (режим API):
```json
{
  "emotion": "anger",
  "intensity": 1,
  "style_vector": {"warmth": -2, "formality": -1, "playfulness": 2, "assertiveness": 0, "expressiveness": 1},
  "detected_style": "sarcasm",
  "explanation": "...",
  "triggers": ["ну конечно", "замечательный", "спасибо огромное"]
}
```

В режиме host возвращает структурированный промпт для хост-LLM, чтобы тот произвёл аналогичный анализ.

### `emotion_de_escalate`

Анализирует эмоции пользователя + переписывает черновик + даёт рекомендации — всё за один вызов.

**Автоматический режим** (по умолчанию): анализирует стиль пользователя, применяет сдвиги деэскалации.
**Режим переопределения**: передайте `target_style` явно.

```json
{
  "user_message": "what the hell is wrong with the delivery?!",
  "draft_response": "Your order should arrive by April 10.",
  "target_style": {"warmth": 1, "formality": 1, "playfulness": 0, "assertiveness": 0, "expressiveness": 0}
}
```

### `emotion_evaluate_dialogue`

Оценивает полный диалог → векторы по каждому сообщению + тренд + риск петли обратной связи.

```json
{
  "messages": [
    {"role": "user", "text": "hello"},
    {"role": "bot", "text": "Hello! How can I help?"},
    {"role": "user", "text": "what the hell is wrong with the delivery?!"},
    {"role": "bot", "text": "Your order #3756 arrives April 10."},
    {"role": "user", "text": "ok I guess I'll wait"}
  ],
  "response_format": "markdown"
}
```

Возвращает таблицу:

| # | Роль | Эмоция | W | F | P | A | E | Стиль |
|---|------|--------|---|---|---|---|---|-------|
| 1 | user | neutral | 0 | -1 | 0 | 0 | 0 | casual |
| 2 | bot | neutral | +1 | 0 | 0 | 0 | 0 | friendly |
| 3 | user | anger | -2 | -2 | -2 | +2 | +2 | aggressive |
| 4 | bot | neutral | 0 | +1 | -1 | 0 | -1 | business |
| 5 | user | neutral | 0 | -1 | 0 | -1 | 0 | resigned |

### Инструменты управления сессиями

Сессии обеспечивают отслеживание эмоционального состояния между ходами разговора.

- **`session_create`** — создать сессию с необязательной пользовательской конфигурацией (целевой аттрактор, скорость сдвига, пороги)
- **`session_get`** — получить состояние сессии: режим, конфигурацию, количество ходов, опционально полную историю
- **`session_reset`** — очистить историю, с возможностью сохранить пользовательскую конфигурацию
- **`session_configure`** — обновить настройки сессии на лету

**Режимы сессий:**
- **Адаптивный** (по умолчанию) — зеркалит стиль пользователя, постепенно сдвигается к положительному аттрактору с каждым ходом
- **Деэскалация** — автоматически активируется при триггерных эмоциях (гнев/отвращение/страх) + высокая напористость или экспрессивность; применяет более сильные корректирующие сдвиги; возвращается к адаптивному режиму после периода остывания

## Установка и настройка

### Требования

- Python 3.11+
- Менеджер пакетов [uv](https://docs.astral.sh/uv/getting-started/installation/)
- API-ключ Anthropic ([получить здесь](https://console.anthropic.com/settings/keys)) — **нужен только для режима API**. Режим host (по умолчанию) работает без него.

> **Важно (macOS / Linux):** GUI-приложения, такие как Claude Desktop, не наследуют PATH из вашего шелла, поэтому `uvx` может быть не найден. Создайте символическую ссылку, чтобы сделать его доступным в системе:
> ```bash
> # Найдите, где установлены uvx и uv
> which uvx
> # Создайте символическую ссылку (пример, скорректируйте путь, если ваш отличается)
> sudo ln -sf ~/.local/bin/uvx /usr/local/bin/uvx
> which uv
> # Создайте символическую ссылку (пример, скорректируйте путь, если ваш отличается)
> sudo ln -sf ~/.local/bin/uv /usr/local/bin/uv
> ```

### Быстрый старт с Claude Code

Самый простой способ использовать этот сервер — с [Claude Code](https://docs.anthropic.com/en/docs/claude-code).

**Шаг 1.** Установите MCP-сервер (одноразовая настройка):

```bash
claude mcp add emotional-deescalation -- uvx emotional-deescalation-mcp
```

> В **режиме host** (по умолчанию) API-ключ не нужен — Claude Code сам выполняет анализ, используя структурированные промпты от сервера.
>
> Для **режима API** (сервер делает собственные вызовы LLM) передайте ключ:
> ```bash
> claude mcp add emotional-deescalation -e ANTHROPIC_API_KEY=sk-ant-your-key-here -e EMOTION_MCP_MODE=api -- uvx emotional-deescalation-mcp
> ```

**Шаг 2.** Запустите Claude Code как обычно:

```bash
claude
```

**Если нужно обновить MCP**
```bash
uv cache clean emotional-deescalation-mcp
claude mcp remove emotional-deescalation
claude mcp add emotional-deescalation -- uvx emotional-deescalation-mcp@latest
```
Готово! Инструменты теперь доступны в вашей сессии Claude Code. Вы можете попросить Claude анализировать сообщения, деэскалировать ответы или оценивать диалоги.

### Claude Desktop

Добавьте в конфигурационный файл Claude Desktop (`claude_desktop_config.json`):

**Режим host (по умолчанию, API-ключ не нужен):**
```json
{
  "mcpServers": {
    "emotional-deescalation": {
      "command": "uvx",
      "args": ["emotional-deescalation-mcp"]
    }
  }
}
```

**Режим API (сервер делает собственные вызовы LLM):**
```json
{
  "mcpServers": {
    "emotional-deescalation": {
      "command": "uvx",
      "args": ["emotional-deescalation-mcp"],
      "env": {
        "ANTHROPIC_API_KEY": "sk-ant-your-key-here",
        "EMOTION_MCP_MODE": "api"
      }
    }
  }
}
```

> **Если `uvx` не найден:** используйте полный путь вместо `"uvx"` (например, `"/Users/you/.local/bin/uvx"`), или создайте символическую ссылку, как описано в разделе [Требования](#требования).

```bash
git clone https://github.com/ilyajob05/emo_bot.git

```

## NLP-сервис (опционально)

Внешний FastAPI-сервис для эмбеддингов предложений и классификации эмоций. MCP-сервер работает без него (использует только spaCy), но NLP-сервис повышает качество за счёт семантического сходства и ML-классификации эмоций.

**Модели эмоций (две специализированные, маршрутизируемые по языку):**
- **Русский:** `cointegrated/rubert-tiny2-cedr-emotion-detection` (~30МБ, 6 эмоций по Экману, F1=0.83)
- **Английский:** `j-hartmann/emotion-english-distilroberta-base` (~82МБ, 7 эмоций по Экману)

**Модель эмбеддингов:** `intfloat/multilingual-e5-base` (эмбеддинги предложений RU + EN)

```bash
# Запуск через Docker
cd nlp_service
docker build -t nlp-service .
docker run -p 8100:8100 nlp-service

# Или переопределение моделей при сборке
docker build \
  --build-arg NLP_EMOTION_MODEL_RU=your/ru-model \
  --build-arg NLP_EMOTION_MODEL_EN=your/en-model \
  -t nlp-service .
```

Настройка через переменные окружения:
- `NLP_SERVICE_URL` — по умолчанию `http://localhost:8100`
- `NLP_EMOTION_MODEL_RU`, `NLP_EMOTION_MODEL_EN` — переопределение моделей эмоций
- `NLP_EMBED_MODEL` — переопределение модели эмбеддингов
